# Workshop Backlog

Use this directory for local workshop exercises, staged learning prompts, and local issues generated by `to-issues`.

Prefer GitHub Issues for public work items. Prefer this directory for exercises that are part of the training material itself or for training runs where learners should inspect issue files in Git.

## File Naming

Use numeric prefixes so the backlog sorts in execution order:

- `001-short-learning-goal.md` for workshop exercises.
- `010-add-basic-validation.md` for generated implementation issues.
- `020-review-agent-output.md` for human review tasks.

## Exercise Template

Use this shape when writing a prompt for a learner and an agent:

```markdown
# 001 - Short Learning Goal

## Goal

What the learner and agent should accomplish.

## Starting Context

What files, concepts, or constraints matter.

## Done When

- Observable result 1
- Observable result 2
- Verification step

## Reflection

What should the learner notice after the agent finishes?
```

## Generated Issue Template

Use this shape when a skill such as `to-issues` publishes local backlog issues:

```markdown
# 010 - Short Implementation Slice

## Type

AFK

## Parent

Reference the parent backlog item, PRD, issue, or source document. Omit this section if there is no parent.

## What to build

A concise description of this vertical slice. Describe the end-to-end behavior, not a layer-by-layer task list.

## Acceptance criteria

- [ ] Criterion 1
- [ ] Criterion 2
- [ ] Criterion 3

## Blocked by

None - can start immediately

## Verification

How the learner or agent should prove the slice works.
```

## Rules for `to-issues`

- Break plans into tracer-bullet vertical slices.
- Prefer AFK issues when the work can be completed without human judgment.
- Mark issues as HITL when they require human decisions, review, or teaching.
- Publish blockers first so later issues can reference them.
- Keep each issue small enough for one focused agent session.
- If using GitHub Issues instead of local files, apply the `needs-triage` label.
